## 0. 序章 — なぜ作ったか(2026-07-05 本人の言葉から)

**学習そのものが、すでにロールプレイングである。**

口が滑らかになる。スペリングが速く、正確になる。間違いが減る。
それは比喩ではなく、本当に起きているステータス上昇だ。LingoSagaはゲームを
発明したのではなく、もともと進行していたゲームに画面を付けただけである。

既存の学習ゲームは、学習の上にゲーム(シナリオ)を被せる。だからシナリオの
出来がすべてで、子供向けの筋書きに大人は飽き、没入できなければ「悔しい」
という感情すら発生しない。

このゲームの主人公は自分自身なので、没入は前提であり演出の成果ではない。
低いスコアの悔しさは本当に自分の実力の話だから消えず、リトライがそのまま
実際の練習になる。**悔しさが練習量に変換される**——これが本作の核である。

だから、最上位ジョブは現実の実績でしか取れない(第5章)。ボス戦は人間が
採点する(8-3)。継続は連続日数ではなく累計で数える(生きて積んだ日は消えない)。
キャラクターを育てているのではない。**キャラクターシートと本人が同一人物**なのだ。

学習だけの話ではない。日々の生活も、ぜんぶロールプレイングである。
続けられない日がある。負ける日も、打ちのめされる日もある。
それでも、どう攻略すれば次は勝てるかを考えて、また挑む。
生きること、学ぶこと——それ自体が、ロールプレイングなのだ。

## 1. 全体アーキテクチャ

```
ChatGPT GPTs（Advanced Voice Mode）
　│ 会話 → 終了時に JSON を出力
　▼
relay.html（セッション記録画面）
　│ JSONを貼り付け → 送信
　▼
GAS（Google Apps Script WebApp）
　│ EXP計算・ランク判定・トリム平均 ← ★計算は全部ここ
　▼
Googleスプレッドシート（データ保存）
　▲
dashboard.html（閲覧・グラフ）
```

**設計原則：**
- GPTsは「採点と記録の出力」だけ。計算はさせない（AIの計算は信用しない）
- EXP・ランク・実力レベルの計算はすべてGASに集約（ルール変更が1箇所で済む）
- HTMLは誰でも共通で使える。個人データは各自のスプレッドシート＋GASに保存
　→ **シェアモデル**：HTMLを配布し、各ユーザーが自分のGAS URLを設定するだけ

## 3. EXP計算式（GASが自動計算）

```
獲得EXP = 分数 × 10 × 難易度倍率 × 出来栄え係数

難易度倍率：Easy 0.8 ／ Normal 1.0 ／ Hard 1.5
出来栄え係数：9パラメーター平均 ÷ 50
```

例：10分 Normal・パラメーター平均55点 → 10×10×1.0×1.1 = **110 EXP**

さぼり減衰（実力レベルのみ・将来実装）：
最終セッションから7日超で表示上「Rusty⚠️」バッジ。ランクとEXPは減らない。

## 5. ジョブツリー（将来実装・設計のみ）

昇格条件は「実力レベル×ランク×クエスト（＋リアル実績）」の複合。
全員が 🧭 Apprentice から始まり、Lv.200＋ランクFで4系統から選んで転職する。

### 共通の昇格レール

| 段階 | 条件 |
|---|---|
| 初期 → 第1職 | 実力Lv.200＋ランクF |
| 第1職 → 第2職 | 実力Lv.400＋ランクD＋転職クエスト（該当話題 Hard 15分×3回） |
| 第2職 → 第3職 | 実力Lv.550＋ランクC＋系統別の腕試しクエスト |
| 第3職 → 最上位 | ★リアル実績のみ★（ゲーム内では絶対に到達不可） |

### 📖 Scholar系：知の道（案内士ルート）

```
📖 Scholar（学者）
　→ 🏮 Local Guide（郷土案内人）
　→ 🎓 Guide Candidate（案内士候補生）※腕試し：模擬面接クエスト
　→ 🏯 Guide Interpreter（通訳案内士）★全国通訳案内士試験に合格★
　→ 👑 Master Guide（伝説の案内人）※ランクS＋実務実績
```

### 🗺️ Traveler系：旅の道

```
🗺️ Traveler（旅人）
　→ 🧳 Wayfarer（渡り歩く者）
　→ 🌏 Globetrotter（世界を巡る者）※腕試し：空港・ホテル・トラブル対応クエスト連続クリア
　→ 🛡️ Pathfinder（開拓者）★リアルの海外一人旅を英語だけで完遂★
```

### 🤝 Diplomat系：交渉の道

```
🤝 Diplomat（外交官）
　→ 💼 Negotiator（交渉人）
　→ 🏛️ Envoy（特使）※腕試し：模擬商談・会議ファシリテーションクエスト
　→ 🌐 Ambassador（大使）★リアルの英語での商談・国際会議・海外取引先対応★
```

### 🎭 Bard系：語りの道

```
🎭 Bard（吟遊詩人）
　→ 📜 Storyteller（語り部）
　→ 🎪 Performer（芸達者）※腕試し：英語で3分間ストーリーテリングクエスト
　→ 🌟 Grand Bard（大吟遊詩人）★リアルの英語でのスピーチ・イベント登壇・動画公開★
```

**リアル融合ポイント**：どの系統も最上位はゲーム内では取れない。
試験合格・海外一人旅・商談・登壇——現実の行動だけが最後の鍵。
ゲームは修行の場、本番は現実。

**設計メモ**：
- 系統をまたぐ「転向」は可（第1職に戻って再スタート。ランクとEXPは維持される）
- Scholar系のStoryteller的要素（蔵の語り・オーラルヒストリー）はBard系と重なる部分があり、
  将来的に「二刀流ジョブ」（例：Guide-Bard＝語り部案内人）を検討する余地あり
- リアル実績の認定は自己申告制。個人利用が前提なので厳密な検証はしない

## 5.5 職業システム（戦闘スタイル称号・毎セッション判定）

「道」（長期キャリア）とは別軸で、**その日の会話の戦い方**をGPTが判定する。
毎回変わってよい。「今日は戦士だった」「今日は僧侶寄りだった」を楽しむ要素。

| 職業 | 戦い方 | 判定のよりどころ |
|---|---|---|
| ⚔️ 戦士 | 多少のミスは気にせずガンガン返す。会話を止めない突破力 | 流暢さ・積極性が高い（文法は多少低くてもよい） |
| 🥋 武闘家 | 短く速い応酬。無駄のない一撃 | 流暢さ・聞き取りが高く、短文中心のテンポ |
| 🧙 魔法使い | 関係詞・仮定法・複雑構文という「呪文」を詠唱する技巧派 | 文法・語彙が突出 |
| 🙏 僧侶 | 柔らかく丁寧、相手を気遣うクッション言葉の使い手 | 場の雰囲気・自然さが高い、丁寧なレジスター |
| 🗡️ 盗賊 | 相手の使った表現をその場で盗んで使い回す吸収型 | 聞き取りが高く、相手のフレーズのエコーが上手い |
| 🎻 吟遊詩人 | 比喩・深みのある表現で物語る | 表現力・自然さが高い |
| 🃏 遊び人 | 冗談・脱線・笑い。スコアはムラがあるが場は沸く | 表現力・積極性が高く、ユーモア発言が多い |
| 💼 商人 | 数字・段取り・交渉に強い実利派 | ビジネス・交渉話題での的確さ |
| 👑 賢者 | 全てを高水準でこなす完成形 | **特別判定**（下記） |

### 賢者の判定（2ルート）

1. **正規ルート**：セッションの9パラメーターがすべて70以上 → そのセッションは賢者判定
2. **遊び人ルート（DQ3オマージュ）**：遊び人判定を累計20回獲得した冒険者は、
   以後さとりのしょ不要で賢者判定の閾値が緩和される（全パラメーター60以上でOK）
   ※累計カウントはGAS側で実装（将来）

### 道との関係

- 道（Scholar系＝知の道 など）＝プレイヤーが選ぶ長期進路。昇格条件で縛る
- 職業＝毎セッションの判定結果。縛りなし・収集要素
- ダッシュボードには「直近の職業」と、将来的には「職業別判定回数」を表示
  （例：戦士12回・僧侶3回・遊び人7回 → 自分の会話の癖が見える化される）

## 7. セットアップ手順（シェア用）

1. 新規Googleスプレッドシートを作成
2. 拡張機能 → Apps Script → `code.gs` を貼り付け
3. 関数 `setup` を1回実行（シートとヘッダーが自動作成される）
4. デプロイ → ウェブアプリ →「全員がアクセス可能」でデプロイ → URLをコピー
5. dashboard.html / relay.html を開き、設定欄にGAS URLを貼って保存
6. GPTsを作成し `english_rpg_v4_prompt.md` をInstructionsに貼る

以上。ユーザーごとに必要なのは**自分のスプレッドシート＋GAS URLだけ**。
